# Phase Transition Rules

Checklist to ensure each project phase is complete before advancing to the next.

## Entry Criteria
- All documents listed for the phase are created and reviewed.
- CHANGELOG and VALIDATION_LOG entries exist for finished work.
- Outstanding risks are captured in `REPO_HEALTH_CHECKLIST.md`.

## Exit Criteria per Phase
- **Phase 0 → Phase 1:** Folder structure + governance docs signed off; no missing templates.
- **Phase 1 → Phase 2:** LabOS core APIs implemented with unit tests and validation notes.
- **Phase 2 → Phase 3:** Data services operational with reproducible ingestion pipelines.
- **Phase 3 → Phase 4:** At least two scientific modules validated; UI shell requirements frozen.

## Approval & Logging
- Each transition logged in `PHASES_OVERVIEW.md` with timestamp, approver, and summary.
- Compliance bot confirms regulatory implications are addressed before progression.
